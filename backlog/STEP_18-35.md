## Outline Construction Pattern

The outline steps follow three distinct patterns:

### Pattern A: Outline with Helper Points (Steps 18-19, 20-23, 26-27)
These steps compute intersection points (PIC, PC) and then create an outline line between them.

```
Inputs (circles, lines, points) 
  → Compute helper points (PICx, PCx) 
  → Create OUTLINEx line connecting them
```

### Pattern B: Simple Outline (Steps 24-25, 28-29, 30-35)
These steps simply connect two existing points with an outline line.

```
Inputs (2 existing points) 
  → Create OUTLINEx line connecting them
```

### Pattern C: Outline with Internal Helper Line (Steps 26-27)
These steps compute a helper line internally, find intercept points, then create the outline.

```
Inputs (circles, lines, points) 
  → Compute internal helper line 
  → Find intercept points (PCx) 
  → Create OUTLINEx line connecting them
```

These steps create the first two outline segments using PII points and PIC points.

#### STEP_18: Outline 1 (PIC4, OUTLINE1)

**Status:** ✅ **IMPLEMENTED**

**Purpose:** Create outline segment 1 from PII1 to PIC4.

**Metrics:**
- Inputs: 3 (PII1, PI4, LCP4PIC12)
- Outputs: 2 (PIC4, OUTLINE1)
- Pattern: A (helper point + outline)

**Implementation:**
- Computes PIC4 as intersection of line(PII1→PI4) with LCP4PIC12
- Creates OUTLINE1 as line from PII1 to PIC4

**Code:**
```typescript
const STEP_18: SixFoldV0Step = {
  id: "step18",
  inputs: [GEOM.PII1, GEOM.PI4, GEOM.LCP4PIC12],
  outputs: [GEOM.PIC4, GEOM.OUTLINE1],
  compute: computeMultiple((inputs, _config) => {
    // lpii1pi4 = line from pii1 to pi4
    const lpii1pi4 = line(pii1.x, pii1.y, pi4.x, pi4.y);
    // pic4 = intersection of lpii1pi4 and lcp4pic12
    const pic4 = lineIntersect(lpii1pi4, lcp4pic12)...;
    // outline1 = line from pii1 to pic4
    const outline1 = line(pii1.x, pii1.y, pic4.x, pic4.y);
    m.set(GEOM.PIC4, pic4);
    m.set(GEOM.OUTLINE1, outline1);
    return m;
  }),
  draw: (svg, values, store, theme) => {
    drawPoint(svg, values, GEOM.PIC4, 2.0, store, theme);
    drawLine(svg, values, GEOM.OUTLINE1, 2.0, store, theme, theme.COLOR_OUTLINE);
  },
};
```

## Mathematical Properties

### Outline Construction Strategy

The six-fold pattern is constructed through a series of outline segments that:

1. **Start at PII1** (the inner intersection point)
2. **Extend outward** through various helper points (PIC, PC)
3. **Connect to circle centers** (CP1-CP4)
4. **Form the complete pattern** with 18 segments

### Symmetry

The outline shows **bilateral symmetry** across the L13/L24 diagonal:
- OUTLINE1 and OUTLINE2 are symmetric
- OUTLINE3 and OUTLINE4 are symmetric
- OUTLINE5 and OUTLINE6 are symmetric
- OUTLINE7 and OUTLINE8 are symmetric
- OUTLINE9 and OUTLINE10 relate to OUTLINE11 and OUTLINE12
- OUTLINE13 and OUTLINE14 are symmetric
- OUTLINE15-OUTLINE18 form the outer quadrilateral

### Geometric Relationships

- **PII1** is the central inner point
- **PIC4, PIC2, PIC1W, PIC1N, PIC23, PIC34** are intersection points on various circles
- **PC1W, PC1N, PC23S, PC23E, PC34E, PC34S, PC3SW** are projection points on circles
- All OUTLINE segments connect these points to form the continuous pattern

---

## Drawing Style

All outline steps use:
- **Line width**: 2.0 (thicker than construction lines which use 0.5)
- **Stroke color**: `theme.COLOR_OUTLINE` (distinct from primary color)
- **Point size**: 2.0 (same as other points)

This makes the final six-fold pattern visibly distinct from the construction geometry.
